w3term - Styling Examples [back to index]]

The w3term library is designed to be easily adaptable to your needs. For using the terminal in your web application, you only need to reference two files in your html file: w3term.js and w3term.css. The css file contains important stylesheets for the terminal to work like a real console.

However, you might want to change the look and feel of your terminal. In order to make this as easy as possible, this page shows you some examples of how to do that. The following terminals are created using an option named stylePrefix, which will cause the terminal to be created using css classes with a certain naming scheme. Using that naming scheme, you can easily override the default styles of the terminal:

Blue Gradient

With a slight text-shadow (see gradient.css)

Background Image

With a slight text-shadow, inset box-shadow, round corners and a custom font. (see bgimage.css)

Gray

(see gray.css)

Black and White

(see bnw.css)

Default Nerdy Green-ish

(without any additional stylesheet)
